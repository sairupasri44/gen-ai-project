# FinRelief AI
## AI Powered Debt Relief and Financial Recovery Platform

FinRelief AI is a full stack web application that helps users facing financial distress
analyze their financial health, predict debt settlement probability, and receive
personalized recovery advice generated by Google Gemini AI.

---

## Features

- User registration and login with duplicate username protection
- Financial dashboard — enter income, debt, and monthly expenses
- Automatic calculation of debt ratio, financial score (0–100), and health label
- 3-condition settlement probability prediction (High / Medium / Low)
- AI-generated financial recovery advice via Google Gemini API
- Full data persistence across 3 relational SQLite tables
- Dark-themed responsive UI with color-coded result badges
- Input validation and error handling on both frontend and backend

## Quick Start

### 1 — Backend Setup

```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

Create a `.env` file inside `backend/`:
```
GEMINI_API_KEY=your_gemini_api_key_here
```

Start the backend:
```bash
uvicorn main:app --reload
```

Backend runs at: `http://localhost:8000`
API docs at: `http://localhost:8000/docs`

### 2 — Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at: `http://localhost:5173`

## Financial Score Formula

```
monthly_debt_payment = debt / 36
leftover             = income - expenses - monthly_debt_payment
score                = (leftover / income) × 100  (clamped 0–100)
```

| Score   | Health    | Settlement           |
|---------|-----------|----------------------|
| 70–100  | Good      | High Probability     |
| 40–69   | At Risk   | Medium Probability   |
| 0–39    | Critical  | Low Probability      |

## Security Notes

- `GEMINI_API_KEY` is loaded from `.env` and never hardcoded
- `.env` is listed in `.gitignore` — never commit it
- All user inputs are sanitized before being injected into Gemini prompts
- All Gemini outputs are HTML-escaped before being returned to the frontend
- Gemini API calls are capped at `max_output_tokens=200` to prevent unbounded consumption
- SQLite queries use parameterized statements — no SQL injection possible
- Duplicate username detection prevents silent account overwrites
